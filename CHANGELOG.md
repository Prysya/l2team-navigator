# Changelog

Все значимые изменения проекта фиксируются в этом файле.

## [1.7.0] - 2026-07-16

### Добавлено
- База рецептов крафта (NG–A grade): 799 рецептов оружия, брони, аксессуаров, сoulshots, эликсиров, ресурсов
- Скрипт сборки `scripts/build-recipes.mjs` — агрегирует данные из трёх JSON-файлов (recipes, items, npc)
- Селекторы в RecipeTab: тип (Оружие/Броня/Аксессуары/Soulshots/Ресурсы/Эликсиры/Прочее) + грейд (NG/D/C/B/A)
- Группировка рецептов в выпадающем списке по подтипу: для брони (Heavy/Light/Robe/Шлемы/Перчатки/Сапоги/Щиты), оружия (Мечи/Дробящее/Кинжалы/Луки/Древковое/Кастеты), аксессуаров (Серьги/Кольца/Ожерелья)
- Три таба в карточке рецепта: «Рецепт», «Куски», «Крафт», «Информация о предмете»
- Дерево компонентов крафта с раскрытием составных материалов
- Таблица мобов для основного куска (первого материала рецепта)
- Динамическая загрузка данных RecipeTab (ленивый импорт RECIPES.json) — основной бандл уменьшен на 80%
- Яндекс.Метрика (ID: 110798252) с отслеживанием просмотров страниц и целей (выбор рецепта, раскрытие босса)
- Утилита `src/utils/metrics.ts` с функциями `hit()` и `goal()` + тесты
- `public/llms.txt` — базовая SEO-разметка для LLM и поисковиков
- Квест «Seed of evil» (уровень 21) в общие квесты всех рас
- 5 Playwright E2E-тестов на RecipeTab

### Изменено
- RecipeTab переписан с нуля: статический `RESOURCES.json` (43 записи) заменён на динамические данные (799 рецептов)
- Табы в карточке рецепта: стилизованы под TabBar (подчёркивание активного таба)
- Соц-сети в бургер-меню: квадратные иконки 44×44 с ховерами (как в хедере)
- Бургер-меню: `overflow-y: auto` на `.burger-body` (разделитель и соцсети не уезжали за экран)
- Quest «Millennium Love» (15 уровень) больше не помечается как обязательный (🔥)
- Quest «Seed of evil»: исправлен ID (171→158), обновлены шаги прохождения
- `chunkSizeWarningLimit` поднят до 5000 KB (RecipeTab загружается лениво отдельными чанками)
- Footer: убран номер версии, добавлено уведомление о Яндекс.Метрике (152-ФЗ)

### Исправлено
- Активный таб сбрасывается на «Рецепт» при смене рецепта (пустой контент при переключении с «Куски»)
- Селектор грейда не применял фильтр для типов «Ресурсы» и «Прочее» (список был пуст)
- Щиты отображались во вкладке «Аксессуары» вместо «Броня → Щиты» (некорректный маппинг subtype)
